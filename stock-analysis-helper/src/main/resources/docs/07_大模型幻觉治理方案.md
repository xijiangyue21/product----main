# 大模型幻觉治理方案

本文记录股票分析助手中针对“日期错误、模拟数据误用、字段编造”等大模型幻觉问题的治理方法。

## 问题背景

股票助手曾出现以下不可信输出：

- 把当前日期回答成旧日期，例如 2024 年 6 月 12 日。
- 将 mock / fallback 行情当成真实行情引用。
- 将模拟历史 K 线计算出的高点、收益率、回撤作为趋势判断依据。
- 在缺少真实基本面数据时，补出 PE、PB、ROE、营收增速等数值。

这些问题的共同点是：模型没有可靠的运行时事实约束，且工具返回值里混入了“可展示但不可分析”的兜底数据。

## 治理原则

1. 运行时事实必须由系统注入，不能让模型猜。
2. 股票助手只采信 live / sina / alpha_vantage 等真实数据源。
3. mock / fallback 数据只能用于页面兜底展示，不能进入 AI 分析上下文。
4. 缺失数据必须显式返回 `--` 或 `unavailable`，不能用假值补齐。
5. 历史分析必须锚定真实 K 线最新交易日，不能用服务器日期推导伪窗口。
6. 旧对话中的错误工具结果必须废弃，不能被后续回答继续引用。

## 关键实现

### 1. 注入运行时日期

在 `StockAssistantController` 中，每次请求都会把当前日期和时区写入用户消息前缀：

- 当前日期来自后端运行时。
- 时区固定为 `Asia/Shanghai`。
- 明确要求模型不要自行猜测日期。

这样模型回答“今天几号”时，依据的是服务端事实，而不是训练记忆或旧上下文。

### 2. 建立 live-only 行情接口

后端新增 live-only 行情路径：

```text
/api/market/live-quote/{code}
```

该接口只返回真实行情源结果。拿不到实时行情时，返回 `source: unavailable` 和 `--` 字段，不再回退到 mock 行情。

股票助手的 `stockQuote` 工具已改为调用该接口。

### 3. 隔离 mock / fallback 数据

主站页面仍可使用 mock / fallback 做展示兜底，但 AI 分析链路禁止使用。

`stockAnalysisContext` 中：

- `quote` 只允许真实行情源。
- `history` 只允许真实 K 线源。
- `fundamentals` 没有真实源时返回 `unavailable`。
- PE / PB / ROE / revenueGrowth 没有真实数据时返回 `--`。

### 4. 改为近期真实 K 线分析

旧的“近一年”历史分析容易受服务器日期、测试数据、模拟 K 线影响。现在改为：

- 使用真实 K 线源。
- 按交易日排序。
- 截取最近 60 个交易日。
- `asOfDate` 使用 K 线最后一个交易日。
- 返回 `analysisWindow: recent` 和 `mockDataUsed: false`。

这避免了用服务器日期生成未来时间窗口，也避免了模拟历史数据进入趋势判断。

### 5. 强化系统提示和工具说明

`system-prompt.txt` 明确要求：

- 当前日期必须以运行时上下文为准。
- 废弃旧对话中的 mock / 测试历史结论。
- `--`、空值、`unavailable` 必须按缺失处理。
- 不得编造价格、估值、收益率、区间高低点、成交量。

工具说明也同步标注：返回值只应来自真实数据源，缺失字段不得补齐。

## 数据可信度约定

| 字段类型 | 可用于分析 | 处理方式 |
| --- | --- | --- |
| `source: sina` | 是 | 可引用 |
| `source: alpha_vantage` | 是 | 可引用 |
| `source: live` | 是 | 可引用 |
| `source: unavailable` | 否 | 说明缺失 |
| `source: mock` | 否 | 禁止进入助手分析 |
| `source: fallback` | 否 | 禁止进入助手分析 |
| 字段值为 `--` | 否 | 说明缺失 |

## 验证方式

### 日期验证

向助手询问：

```text
今天日期几号？只回答日期。
```

期望：回答后端运行时注入的日期，而不是旧日期或模型猜测日期。

### 数据源验证

请求：

```text
GET /api/market/ai-context/600519
```

关键字段应满足：

```json
{
  "analysisWindow": "recent",
  "mockDataUsed": false,
  "dataSources": {
    "quote": "sina",
    "history": "sina",
    "fundamentals": "unavailable"
  },
  "pe": "--",
  "pb": "--",
  "roe": "--"
}
```

### 历史窗口验证

`historyStartDate` 和 `historyEndDate` 应来自真实 K 线交易日：

- 不使用服务器当前日期作为结束日期。
- 不返回模拟高点或模拟收益率。
- `historyDays` 约为 60 个交易日。

## 代码落点

- `StockAssistantController`：注入运行时日期和时区。
- `StockMarketTool`：股票工具调用 live-only 行情接口。
- `MarketController`：暴露 live-only quote 接口。
- `MarketService`：构造 AI 分析上下文、禁用 mock、近期 K 线分析。
- `system-prompt.txt`：约束模型不得猜日期和编造数据。

## 后续建议

- 为真实基本面数据接入独立可信源，不要使用本地静态估值。
- 给 `ai-context` 增加自动化测试，断言不出现 `mock`、`fallback` 和伪估值。
- 对每个工具返回值保留 `source` 字段，前端和助手都应展示或使用该字段判断可信度。
- 如果真实行情源不可用，应直接提示“数据暂不可用”，不要降级生成投资结论。
